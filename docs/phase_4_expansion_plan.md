# Phase 4: Expansion & Commercialization 🚀

> **撰写时间**: 2026-03-30 
> **阶段定位**: 在完成「Phase 3 协作闭环」并开启核心功能内测（Beta）期间，并行启动的**三大扩展性护城河战役**。目标是打通无上限的大规模数据存储、占领移动端流量入口，并搭建 SaaS 计费模型以实现造血。

---

## 一、 对象存储重构 (MinIO / S3 Integration)

**🎯 战略目标**：彻底解放当前服务器的本地硬盘 I/O 与容量瓶颈，支持海量附件、高频历史快照（Snapshot）与结构化多媒体资产的高可用、多端加速与容灾。 

### 1.1 前置预签名网络架构 (Presigned URL)
- **客户端直传**：客户端直接向 Rust 后端获取 MinIO/AWS S3 的带时效上传凭证 (Presigned PUT URL)。客户端携此凭证将大文件（视频、巨幅长图等）**直接打向对象存储集群**，彻底终结 Rust 网关节点被流式传输打满带宽的隐患。
- **动态寻址**：通过 CDN 在全球范围加速图片获取 (Presigned GET URL)，笔记中的 `![](...)` 链接依然只保存短哈希，展示时实时换取有时效的访问链。

### 1.2 数据热迁移与双轨制
- 维护现有的 `LocalFileStorage` 特性，允许 Community Edition (社区免费容器版) 继续使用本地卷。
- 研发 `S3FileStorage` 直连驱动，并构建平滑无缝的数据上云热迁移脚本。

### 1.3 存储池治理与隔离防线
- **Vault ID 命名空间**：基于团队 `Vault_ID` 在云端划分强隔离目录流。即使 Bucket 数据泄漏，依然无法跨租户越权。
- **资源生命周期 (Lifecycle)**：结合 Slash 的系统级回收站机制（Trash Lifecycle），利用 S3 Object Lifecycle 原生能力：30天沉睡进入冷归档（Glacier），60天彻底从物理宇宙抹除。

## 二、 多租户工作区引擎 (Multi-Tenant Workspace Switcher)

**🎯 战略目标**：打破目前客户端单实例单连接的局限，提供类似 Slack / Figma 的「一键切流」高维体验，为企业级商业化和横向病毒式传播奠定最核心的物理与 UI 基础。

### 2.1 客户端状态机重构
- **多实例并发连接**：重写前端 `SyncService` 和 `AutoSyncManager`，从单例控制转为集群监控。允许用户在同一台设备上，同时登录绑定并监听 `Team_Alpha_Project` 和 `Team_Beta_HQ` 两个不同远程服务器/频道的本地文件夹。
- **全局状态隔离**：确保切换租户工作区时，Redux 或 Zustand 状态树实现秒级卸载与重装，避免上一个团队的敏感数据（如成员列表、权限控制字）在下一个团队中发生状态泄漏。

### 2.2 跨空间极速切流 (UI 交互)
- **左侧边栏顶层下拉菜单**：在界面最核心位置引入强视觉层级的「当前空间切换器」。
- **零延迟感受**：当鼠标点击切换团队时，无需经历传统的「登出、清缓存、重新输入密码登录的 Loading」，利用已就绪的 Keychain 持久化账密或多令牌池结构，实现毫秒级 UI 重渲染。

## 三、 移动端原生化 (Mobile App: iOS/Android)

**🎯 战略目标**：构建 Slash 协作生态的全平台化最后一块拼图——占领口袋级生产力空间，实现随时随地的多端同步、随手写与富文本灵感捕捉。

### 3.1 跨端技术选型攻坚
- **Tauri v2 Mobile (Rust Core + React Native frontend)**：重度复用当前我们引以为傲的 Rust 同步核心算法（Task Bypass / CRDT / Hash 验证机制）与底层能力，将已有代码全量编译打包至 iOS/Android（IPC 调用取代网络请求）。
- **离线优先 (Local-First Mobile)**：移动网络有着致命的高延迟与断网场景。我们要依靠 SQLite 基座打硬仗——所有修改优先保存本地 SQLite，当移动信号恢复时再利用我们在 P3 中完美验证的、精巧无比的「行级哈希锚定」秒级合并至云端。

### 3.2 移动端 UX 范式大革命
- **富文本沉浸适配**：将 ProseMirror / TipTap 的菜单条重制为底层悬浮式键盘扩展行机制（类似 Notion Mobile）。彻底隐藏复杂的侧边栏（FileTree），采用深色沉浸的全屏聚焦书写态。
- **段落批注的视觉映射**：原本可以在桌面端舒展开的侧边「批注/评论流」，需全部折叠进入 Bottom Sheet 层叠抽屉。划词交互重构为针对拇指热区优化的触屏放大镜长按激活，极度考验 UI 打磨细节。

## 四、 商业化变现与订阅制 (Commercialization)

**🎯 战略目标**：明确产品从 “Geek 玩具”向“工业级 SaaS”蜕变。构建一整套坚固、安全的计费墙、权限下发管线，为 Slash 的持续迭代构建正向的商业循环。

### 4.1 产品的楚河汉界定位
- **Community Edition (社区版)**：持续开源免费。支持本地离线知识图谱、个人多端同频同步（Docker All-in-one 容器支持）。培养基本盘与品牌自传播效应。
- **Team Edition (企业托管版)**：基于我们的中央调度云构建，启用「全量协作共识引擎」、「RBAC 角色颗粒度隔离」、「无限 Snapshot 快照线反黑客追溯」以及对象存储。全面按量计费收费。

### 4.2 阶梯配额池与限流模型 (Quota System)
- **多维计费因子**：按最高可绑定成员阶梯计价（如最大 5 席 / 最大 15 席 / 最大无限席）；按 Team Space / Project 存放容量设置墙。
- **限流熔断模块**：引入高吞吐量的高并发内存库（如 Redis）开发基于令牌桶（Token Bucket）的请求熔断组件。严防自动化越权拉拽刷爆流量带宽（尤其防范 API Key 的滥用与爬虫）。

### 4.3 支付全自动流转体系
- **全球化网关集成**：基于 Stripe (境外) 与 微信支付/支付宝 (境内) 的 webhook 生命周期构建核心的 `subscription_event` 事件。
- **瞬时权限开通与降级**：
    - 即时开通：支付成功回调的 100ms 内，Rust 后端即刻拉起 Team 的创建、Owner 委任以及邀请码额度的颁发。
    - 欠费降级：当订阅到期，非暴力强删，而是自动触发并注入 `Read-Only` 降级引擎锁死全局修改。直至续费后自动解除只读红线。

## 📅 阶段实施里程碑 (在目前内测期间的并行管线)

1. **里程碑 1 (T+0开始)** 攻坚并补全 **MinIO 对象存储生态** —> 它是客户端存放多媒体并在移动端实现即时云拉取的基础设施，更是后续我们计算商业化存储配额容量的硬物理前提。
2. **里程碑 2 (T+5启动)** 重构桌面端切流体验，构建 **多租户工作区切换器 (Workspace Switcher)** —> 只有前台明确了 Team 的物理界限并能极速一键切换，后续的订阅流水单才能逻辑自洽地挂载至独立的 Team Space。
3. **里程碑 3 (T+15接入)** 基于已成熟的账号与 Vault 系统接入 **Stripe / 微信支付沙盒** —> 开始向内测活跃团组试探性验证收费壁垒。
4. **里程碑 4 (T+30并行)** 调动核心重兵啃下 **Tauri Mobile** UX/UI 原生化的骨头，利用我们在桌面端稳定下来的离线 SQLite "Task Bypass" 引擎降维打击彻底占领多端领地。
