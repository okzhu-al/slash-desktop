## 8. P3-6: 知识图谱校准与 GhostLink 精度优化

我们成功实现了更加精准的 GhostLink (相似笔记关联推荐) 算法，全面超越了原先单一粗糙的余弦相似度匹配：

- **文件修改**：[ghostlink.rs](file:///Users/junior/Projects/slash/apps/desktop/src-tauri/src/core/ai/skills/ghostlink.rs)
- **改进点**：
  1. **混合相似度评分计算 (Hybrid Score)**：
     - **标签相似度 (Tag Similarity)**：引入了 `parse_tags` 和 `jaccard_similarity`。通过计算源笔记与候选笔记的 `user_tags` 及 `ai_tags` 交并比（Jaccard 相似度），如果共享较多标签，推荐评分会获得高置信度加成。
     - **图拓扑相似度 (Graph Topology Similarity)**：引入了二阶邻接图查询。利用 `links` 表检索出全库所有的双向引用关系，在内存中构建出全局邻居集缓存 `neighbors_map`（避免 N+1 查询造成的性能衰退）。通过计算两篇笔记在图谱拓扑中共享的邻居笔记 Jaccard 相似度来修正匹配精度。
     - **合成加权**：最终推荐评分使用混合加权公式：`0.5 * sim_vector + 0.3 * sim_tag + 0.2 * sim_graph`。
  2. **动态自适应过滤阈值 (Dynamic Threshold)**：
     - 引入了 `calculate_dynamic_threshold` 策略。过滤阈值不再是静态的 `0.60`，而是根据笔记的文本大小进行动态校准。
     - 对于低于 200 字节的超短笔记，加征惩罚阈值（防高熵噪声）；对于超长笔记，适当降低准入门槛。最终阈值被夹逼限制在 `[0.30, 0.95]` 区间内。
  3. **单元测试保障 (Unit Tests)**：
     - 编写了 `test_parse_tags`、`test_jaccard_similarity`、`test_calculate_dynamic_threshold` 三组完整的单元测试用例，覆盖全部新加入的数学和解析算法，并顺利通过。
- **验证结果**：
  - 执行 `cargo test --lib core::ai::skills::ghostlink` 四个单元测试（包含原有的 `test_cosine_similarity`）全部成功通过。

## 10. Sidecar 安装包资源路径适配与 Mac/Windows 兼容性增强

### 隐患及根本原因分析

我们在安装完 macOS 的 `.dmg` 包后，发现即使更新了 `apps/python-sidecar/version.json` 到 `1.0.2`，主程序运行后释放到 `Application Support/Slash/sidecar` 下的 Sidecar 仍然停留在 `1.0.0` 版本。
经过对 [sidecar.rs](file:///Users/junior/Projects/slash/apps/desktop/src-tauri/src/core/sidecar.rs) 寻找 bundle 资源机制的深度排查，发现了如下问题：
1. **macOS 打包路径结构不符**：
   在 `tauri.conf.json` 中，Sidecar 目录被配置为 `"resources": ["binaries/*"]`。Tauri v2 打包时，会直接将 `binaries/*` 匹配的子目录（即 `slash-sidecar-aarch64-apple-darwin`）放置在 `.app/Contents/Resources/` 根目录下，**不会**生成中间层 `binaries` 目录。
   然而，原 Rust 代码中的路径匹配逻辑仅查询了：
   `Slash.app/Contents/Resources/binaries/slash-sidecar-aarch64-apple-darwin`
   这导致 `find_bundle_sidecar_dir` 和 `find_sidecar_binary` 对安装后的生产包定位失败（返回 `None`）。
2. **导致结果**：
   - 首次启动由于未能找到 Bundle 资源，而直接跳过了释放；
   - 后续版本比对时由于 `bundle_sidecar_ver` 默认返回 `0.0.0`（因找不到目录），导致其永远小于已安装的 `1.0.0`，静默升级机制完全失效。

### 修复与兼容性增强实现

我们在 [sidecar.rs](file:///Users/junior/Projects/slash/apps/desktop/src-tauri/src/core/sidecar.rs) 中同时修复并强化了 `find_sidecar_binary` 和 `find_bundle_sidecar_dir` 的寻找路径：
- **增加 macOS v2 生产包备选路径**：在保留原路径查找的基础上，新增了对无 `binaries` 中间路径的查找：`exe_dir.parent().join("Resources").join(binary_name)`。
- **兼容 Windows Flat Layout 路径**：支持同级直接存在 `binaries/{binary_name}` 布局。

### 验证情况
- 运行 `cargo check` 编译成功，无任何编译警示。
