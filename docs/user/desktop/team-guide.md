# 团队与同步指南

> 状态：当前
> 受众：用户
> 负责：Slash
> 最后审阅：2026-05-31

本文介绍 Slash Desktop 中的个人同步、团队空间、权限、评论批注和版本协作。团队能力需要连接 Slash Server。

## 基本概念

Slash 可以完全本地使用。连接 Slash Server 后，会增加两类能力：

- 个人同步：在多台设备之间同步自己的 Vault。
- 团队协作：和团队成员共享指定目录、文件、评论、批注、版本和任务状态。

Slash 的团队协作是本地优先、异步同步的轻协作模型，不是多人同时编辑同一段文字的实时协作编辑。

## 连接 Server

1. 打开 Settings。
2. 进入同步或 Server 连接设置。
3. 输入服务器地址。
4. 首次连接时输入服务端日志中的 Access Code。
5. 按界面提示设置 PIN。

后续新设备通常使用 PIN 连接。如果忘记 PIN，可以按界面提示重新生成配对码，并从 Server 日志中查看新的 Access Code。

生产环境建议使用 HTTPS 地址，例如：

```text
https://slash.example.com
```

局域网测试可以临时使用 HTTP。macOS 正式安装包连接公网服务时应使用 HTTPS。

## 个人同步

个人同步用于多设备使用同一个 Vault。

常见同步内容包括：

- Markdown 笔记。
- 图片、视频、PDF 和其他资产。
- 笔记移动、重命名和删除。
- 任务 checkbox 状态。

同步失败时，本地内容仍保留在 Vault 中。恢复网络或重新连接 Server 后，可以再次同步。

## 团队空间

团队空间用于共享团队知识库。团队管理员可以创建团队、邀请成员，并把目录授权给不同成员。

团队空间中常见操作包括：

- 创建团队。
- 邀请成员加入。
- 将个人目录 Promote 到团队空间。
- 为团队目录设置成员权限。
- 查看团队目录、团队文件和团队活动。
- 管理团队回收站和存储空间。

Promote 到团队空间前，建议先确认目录结构、命名和内容范围，避免把仍在整理中的私人资料误共享。

## 权限角色

Slash 团队权限分为全局角色和目录角色。

| 角色 | 含义 |
| --- | --- |
| Admin | 管理团队、成员、维护模式、团队设置和高风险操作。 |
| Owner | 管理被授权目录，通常可以编辑、同步和维护该目录内容。 |
| TeamMember | 参与被授权目录的编辑和同步。 |
| Observer | 查看允许可见的内容，不作为普通编辑者同步目录。 |

最终权限由 Server 校验。客户端里的只读提示是体验层保护，不替代服务端权限判断。

## 评论、批注和版本

团队文件可以使用轻协作能力：

- 评论：围绕文件内容讨论。
- 批注：对具体内容区域留下反馈。
- 版本历史：查看历史版本，必要时回退。
- 协作事件：查看团队成员对文件的操作活动。

这些能力依赖文件身份，而不是单纯依赖路径。因此文件重命名后，评论、批注和版本仍应跟随同一个文件。

## 编辑锁

对于需要协作保护的团队文件，Slash 会使用编辑锁减少覆盖风险。

- 如果你获得锁，可以编辑并同步。
- 如果文件被其他成员持锁，界面会提示当前状态。
- 离线或网络异常时，Slash 会更保守地保护编辑，避免误覆盖他人内容。

这不是实时多人编辑，而是为了让异步同步下的编辑责任更清楚。

## 任务同步

团队文件中的任务 checkbox 可以更快同步到 Server。点击完成或未完成后，其他成员通常能比普通保存周期更快看到状态变化。

如果同一行任务内容被多人同时修改，Server 会优先保护一致性，不会猜测应修改哪一行。

## 维护模式

当 Admin 整理团队目录、移动大量文件或执行维护操作时，可能开启维护模式。维护模式期间，普通成员的部分同步或编辑操作可能会受限。

看到维护提示时，建议等待管理员完成维护后再继续大量编辑。

## 安全建议

- 团队 Server 应使用 HTTPS。
- 重要数据应保留本地 Vault 备份。
- Promote 目录前先检查是否包含私人内容。
- 不要把 PIN、Access Code 或团队账号密码发给无关人员。
- 遇到同步异常时，先不要手动删除本地文件；保留现场并联系管理员或反馈 Issue。

## 常见问题

### 团队协作是不是实时协同编辑？

不是。Slash 当前是本地优先、异步同步、权限控制、评论批注、版本和编辑锁模型。

### 为什么我能看到某个目录，但不能同步或编辑？

你可能是 Observer，或者只拥有查看权限。可见范围和同步范围不是同一件事。

### 被移出团队后会怎样？

客户端会停止继续访问对应团队空间。已在本地的内容如何处理，按应用提示和团队策略执行。

### 同步失败会丢本地笔记吗？

正常情况下不会。Slash 的用户内容首先是 Vault 中的 Markdown 和资产文件。同步失败时，应先保留本地现场，再排查连接、权限和 Server 状态。
