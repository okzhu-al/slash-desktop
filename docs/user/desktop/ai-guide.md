# AI 使用指南

> 状态：当前
> 受众：用户
> 负责：Slash
> 最后审阅：2026-05-30

Slash AI 是本地优先的辅助能力。它可以帮助你总结、打标签、处理选中文本、发现潜在关联和辅助归档，但不会替你静默移动、删除或改写笔记。

## AI 能做什么

- 生成摘要建议。
- 生成标签建议。
- 为笔记推荐标题。
- 对选中文字执行 AI Skill。
- 推荐潜在相关笔记。
- 辅助语义搜索和深度搜索。
- 推荐笔记应该归入的文件夹。

AI 结果默认是建议。只有你确认采纳后，才会写入用户标签、摘要、标题或文件位置。

## Provider 类型

Slash 把 AI 分为生成模型和嵌入模型。

### 生成模型

生成模型用于摘要、标签、AI Skill、分类、关联理由等文本生成任务。

你可以选择：

- 本地 Ollama。
- 在线 OpenAI-compatible Provider。

在线 Provider 需要你自己配置 API Key、Base URL 和模型名。API Key 会保存在系统 Keychain 中，不以明文保存在普通配置文件里。

### 嵌入模型

嵌入模型用于语义搜索、GhostLink 候选召回、文件夹向量和 RAG 索引。

当前嵌入固定使用本地 Ollama 的 `bge-m3`。即使你把生成模型切到在线 Provider，语义索引仍依赖本地 `bge-m3`。

## 快速开始：本地 Ollama

1. 安装并启动 Ollama。
2. 在 Ollama 中准备一个生成模型。
3. 打开 Slash 设置页。
4. 进入 AI 设置。
5. 选择本地 Provider。
6. 设置 Ollama host、port 和生成模型。
7. 确认连接状态正常。

如果模型列表加载失败，请确认 Ollama 正在运行，默认端口通常是 `11434`。

## 快速开始：在线 Provider

1. 打开 Slash 设置页。
2. 进入 AI 设置。
3. 新增在线 Provider。
4. 填写 Base URL、模型名和 API Key。
5. 保存并激活该 Provider。

注意：在线 Provider 会把相关文本发送到你配置的服务商。请确认你信任该服务商，并理解对应服务的隐私和计费规则。

## 文件夹级 AI 配置

你可以为某个目录设置不同的生成 Provider 或模型。Slash 会从当前笔记所在目录向上查找 `.slash-folder.yml`。

示例：

```yaml
ai:
  provider: online
  model: gpt-4.1-mini
```

这只影响生成任务，不会改变本地嵌入模型。

## AI Skill

AI Skill 通过 `/` 菜单触发。

基本流程：

1. 输入 `/`。
2. 选择 AI Skill。
3. 在编辑器中选择一段文字。
4. 确认执行。
5. 等待结果插入或展示。

AI Skill 适合翻译、改写、总结、解释、生成结构化内容等任务。执行期间编辑器会显示处理状态。

## 自定义 Skill

自定义 Skill 是 Vault 级配置，保存在 `.slash/skills/` 中。你可以在设置页创建和编辑。

常见用途：

- 翻译成英文。
- 把会议记录整理成行动项。
- 把长段落改写为提纲。
- 用指定语气润色文本。

如果你使用 `raw_prompt` 类型，Slash 会按原始提示词执行。普通技能会把选中文本作为用户内容处理，降低提示词注入风险。

## 摘要、标签和标题建议

Slash 可以为笔记生成摘要、标签和标题建议。

- 生成内容会先作为 AI 建议展示。
- 接受标签后，标签会写入用户标签和 frontmatter。
- 接受标题后，Slash 可能会重命名文件。
- 拒绝建议后，该建议会从当前建议列表中移除。

如果你手动修改了摘要或标签，用户修改优先。

## GhostLink

GhostLink 会根据语义相似度推荐潜在相关笔记。

你可以：

- 查看候选关联。
- 请求 AI 解释为什么相关。
- 接受关联。
- 忽略不想再看到的推荐。

GhostLink 候选来自本地语义索引；关联理由可能使用你配置的生成 Provider。

## 智能分类

智能分类用于推荐笔记应该归到哪个文件夹。

它会参考：

- 笔记内容。
- 摘要和标签。
- 当前 Vault 中真实存在的目录。
- 文件夹语义信息。

分类只是建议。只有你确认后，Slash 才会移动文件。

## 语义搜索

语义搜索依赖本地 embedding 索引。新笔记或刚修改的笔记可能需要等待后台索引完成后，才能被语义搜索充分命中。

如果语义搜索没有结果：

1. 确认 Ollama 正在运行。
2. 确认 `bge-m3` 可用。
3. 在维护入口检查或重建 AI 索引。
4. 等待后台索引完成。

## 隐私与安全

- 本地 Ollama 模式下，生成和嵌入都在本机运行。
- 在线 Provider 模式下，生成任务会发送相关文本到你配置的服务商。
- API Key 存储在系统 Keychain 中。
- AI 日志不应包含完整 API Key 或完整长文本。
- AI 失败不会阻止你继续编辑、保存、打开和搜索。

## 常见问题

### AI 没有响应

- 检查 AI 设置里的 Provider 是否可用。
- 本地模式确认 Ollama 已启动。
- 在线模式确认 API Key、Base URL、模型名和服务余额。

### 语义搜索不准

- 等待后台索引完成。
- 确认本地 `bge-m3` 可用。
- 对很短的笔记，语义效果可能不明显。

### 标签或摘要不满意

- 可以拒绝建议。
- 可以手动编辑用户标签或摘要。
- 可以换用更适合你内容语言的生成模型。

### 在线 Provider 会影响本地索引吗

不会。当前在线 Provider 只影响生成任务，语义索引仍使用本地 `bge-m3`。
