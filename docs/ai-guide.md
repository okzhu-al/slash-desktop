# Slash AI 功能用户指南

Slash 内置了一套完整的本地优先 AI 引擎，覆盖笔记的自动摘要、智能标签、重命名、语义搜索、知识图谱推理和 PARA 智能归档等场景。所有 AI 处理均在本地完成（Ollama）或通过你配置的在线模型运行，笔记内容不会发送到未授权的第三方服务。

## 1. 快速开始

### 最低要求

| 组件 | 说明 |
|------|------|
| **Ollama**（推荐） | 本地运行 LLM，需安装 [ollama.com](https://ollama.com) |
| **生成模型** | 如 `qwen2.5:7b`、`llama3.1:8b`、`gemma2:9b` 等 |
| **嵌入模型** | 固定使用 `bge-m3`（自动下载） |

### 30 秒启动

1. 安装 Ollama 并启动服务
2. 打开 Slash → 设置 → AI 标签页
3. 选择「本地 (Ollama)」模式
4. 选择或下载一个生成模型（如 `qwen2.5:7b`）
5. 确认连接状态显示 ✅
6. 开始编辑笔记，AI 功能将自动激活

## 2. AI 提供者配置

Slash 支持两种 AI 提供者模式：

### 本地模式（Ollama）

使用本地部署的 Ollama 服务，数据完全不出本机。

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| Host | `localhost` | Ollama 服务地址 |
| Port | `11434` | Ollama 服务端口 |
| 生成模型 | — | 用于标签/摘要/重命名 |

> 💡 如果模型未下载，可在设置页面点击下载按钮，进度条会实时显示。

### 在线模式

支持 OpenAI 兼容的 API 服务（Gemini、OpenAI、DeepSeek、通义千问等）。

| 配置项 | 说明 |
|--------|------|
| API Key | 填入你的 API 密钥 |
| Base URL | API 端点地址 |
| Model | 选择可用模型 |

> ⚠️ 在线模式下，笔记内容会发送到对应的 API 服务。模型密钥通过系统 Keychain 安全存储，不存于明文文件中。

### 嵌入模型（固定，本地 Ollama）

> 🔑 **重要**：无论你选择本地还是在线生成模型，嵌入（Embedding）始终使用本地 Ollama 的 `bge-m3` 模型。这意味着你**必须安装并运行 Ollama**，即使生成模型使用在线 API。

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 嵌入模型 | `bge-m3` | 固定，不可更改 |
| 运行方式 | 本地 Ollama | 始终本地运行 |

## 3. 核心 AI 技能

Slash 内置三个核心 AI 技能，均为**手动触发**：

### 📝 智能摘要（Summary）

- **功能**：为笔记生成简洁的中文/英文摘要
- **触发方式**：在编辑器顶部工具栏点击「摘要」按钮
- **输出位置**：摘要写入 YAML frontmatter 的 `summary` 字段
- **智能跳过**：内容未变化时自动跳过，不重复调用

### 🏷️ 智能标签（Tagging）

- **功能**：基于笔记内容自动提取语义标签（3-5 个）
- **触发方式**：在编辑器顶部工具栏点击「标签」按钮
- **输出位置**：标签存储在数据库 `ai_tags` 字段
- **RAG 增强**：自动注入已有标签库作为参考，保证标签用词一致性
- **反馈机制**：你可以接受或拒绝每个标签（详见 [AI 反馈系统](#9-ai-反馈系统)）

### ✏️ 智能重命名（Smart Rename）

- **功能**：基于内容为笔记生成更有意义的文件名
- **触发方式**：在编辑器顶部工具栏点击「重命名」按钮
- **安全保障**：自动过滤路径遍历字符（`/`、`\`、`..`、`:` 等），防止注入攻击
- **冲突处理**：如目标文件名已存在，自动添加 `_1` 后缀

## 4. AI 浮动工具栏（Magic Wand）

在编辑器中选中一段文本，会浮现一个 **✨ AI 工具栏**：

1. 点击 ✨ 按钮展开技能选择器
2. 选择一个自定义技能（Custom Skill）
3. 选中的文本作为输入发送给 AI
4. AI 返回结果将**替换**选中文本

支持流式输出——你可以实时看到 AI 生成的内容逐字显现。

## 5. 自定义技能（Custom Skills）

你可以通过 YAML 定义自己的 AI 技能，用于翻译、改写、提问等任何场景。

### 创建方式

设置 → AI → 「自定义技能」标签页 → 点击「新建」

### YAML 格式示例

```yaml
name: 翻译为英文
description: 将选中的中文文本翻译为英文
system_prompt: |
  You are a professional translator.
  Translate the following Chinese text to natural, fluent English.
  Output ONLY the translation, no explanations.
user_prompt: |
  请翻译以下内容：
  {content}
temperature: 0.3
max_tokens: 2048
stream: true
```

### 关键参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `name` | 技能名称（显示在菜单中） | — |
| `description` | 技能描述 | — |
| `system_prompt` | 系统提示词（定义 AI 角色） | — |
| `user_prompt` | 用户提示词，`{content}` 占位符被替换为输入文本 | — |
| `temperature` | 生成随机性（0.0-1.0，越低越确定） | `0.7` |
| `max_tokens` | 最大输出长度 | `2048` |
| `stream` | 是否流式输出 | `true` |

> 💡 自定义技能在 AI 浮动工具栏中可用——选中文本后点击 ✨ 即可选择执行。

## 6. 智能搜索

Slash 提供 4 层搜索能力，统一入口为 `Cmd+K`（或 `Ctrl+K`）：

### 🔍 快速搜索（Quick Search）

- **触发**：`Cmd+K` 弹出搜索栏
- **机制**：全文检索（FTS5） + 文件名匹配
- **速度**：毫秒级

### 🧠 语义搜索（Semantic Search）

- **触发**：搜索栏 → 「AI」标签页
- **机制**：将查询转化为向量，与笔记嵌入做余弦相似度匹配
- **要求**：需要先完成嵌入索引构建

### 🔮 HyDE 搜索

- **触发**：搜索栏 → 「AI」标签页 → HyDE 模式
- **机制**：先让 LLM 生成一篇假设文档（Hypothetical Document），再用该文档的嵌入做语义搜索
- **优势**：对抽象或模糊查询效果更好

### 🧬 深度搜索（Deep Search / RAG）

- **触发**：搜索栏 → 「AI」标签页 → Deep Search
- **机制**：语义检索 + LLM 综合推理（RAG 架构）
- **输出**：生成一段结构化的分析报告，引用具体笔记内容
- **参数**：确定性输出（temperature = 0.0）

## 7. Ghost Links（幽灵链接）

Ghost Links 是 Slash 独有的知识发现功能——AI 自动发现你的笔记之间潜在的关联关系。

### 工作原理

1. 编辑笔记时，AI 分析内容语义
2. 与 Vault 中其他笔记的嵌入做相似度对比
3. 发现高相似度的笔记对，推理出关系类型
4. 在知识图谱（Graph View）中以虚线展示

### 关系类型

| 类型 | 含义 |
|------|------|
| `supports` | A 支持 B 的论点 |
| `contradicts` | A 与 B 存在矛盾 |
| `extends` | A 扩展了 B 的内容 |
| `related` | A 与 B 在主题上相关 |

### 操作

- **接受**：将 Ghost Link 转化为正式的 WikiLink
- **忽略**：将该关系加入黑名单，不再推荐
- **查看原因**：AI 会解释为什么认为两篇笔记相关

## 8. PARA 智能归档

Slash 实现了 PARA 方法论的智能归档系统，帮助你将 Inbox 中的笔记自动分类到 Projects/Areas/Resources/Archives。

### 触发条件

- 笔记必须位于 `00_Inbox/` 目录下
- 笔记内容需足够成熟（>50 字符有效内容）
- 不含未完成任务（`- [ ]`）

### 分类流程

```
笔记内容 → 启发式匹配 → 向量相似度 → LLM 决策 → 推荐目标目录
```

1. **启发式层**：基于关键词和文件名快速匹配
2. **向量层**：计算与已有目录笔记的嵌入相似度
3. **LLM 层**：当前两层不确定时，调用 LLM 进行最终判断

### 使用方式

- 打开 Inbox 中的笔记，底部状态栏会显示归档建议
- 点击接受即可将笔记移动到推荐目录
- 非 Inbox 笔记会自动显示「Local Graph」而非归档建议

## 9. AI 反馈系统

Slash 内置了完整的反馈闭环，帮助你评价 AI 输出质量：

| 操作 | 说明 |
|------|------|
| **接受标签** | 将 AI 标签移至用户标签（从 `ai_tags` → `user_tags`） |
| **拒绝标签** | 从 AI 标签中移除 |
| **接受摘要** | 将 AI 摘要保存为用户摘要 |
| **编辑摘要** | 修改后保存为用户版本 |
| **标签纠正** | 手动添加的标签会被记录为训练数据 |

所有反馈数据存储在本地 `ai_feedback` 表中，包含：输入快照、模型名称、动作类型、原始输出和用户修正，为后续模型微调提供高质量训练三元组。

## 10. Embedding 管线

嵌入管线是语义搜索和知识发现的基础设施，负责将你的笔记转化为向量表示。

### 自动行为

- 编辑笔记后自动触发嵌入更新（后台异步执行）
- 内容未变化时跳过（基于 SHA-256 增量检测）
- 笔记删除后自动清理孤儿嵌入

### 手动管理

设置页面或通过快捷命令可以：

| 操作 | 说明 |
|------|------|
| 查看统计 | 已嵌入笔记数、失败数、版本信息 |
| 重建全部 | 清除并重新生成所有嵌入（耗时较长） |
| 重建单篇 | 强制重新生成指定笔记的嵌入 |
| 暂停管线 | 临时停止后台嵌入处理 |
| 重试失败 | 重新处理之前失败的笔记 |
| 清除缓存 | 删除所有嵌入数据 |

> ⚠️ 嵌入模型固定使用 `bge-m3`。如果更换模型版本，需要执行「重建全部」以保证向量维度一致。

## 11. 常见问题

### Q: AI 功能没有任何响应？

1. 检查设置 → AI → 连接状态是否为 ✅
2. 本地模式：确认 Ollama 服务正在运行（终端执行 `ollama list`）
3. 在线模式：确认 API Key 正确且余额充足

### Q: 标签/摘要质量不理想？

- 尝试更换更大的生成模型（推荐 7B+ 参数）
- 使用反馈系统标记不准确的结果
- 对于中文内容，推荐 `qwen2.5` 系列模型

### Q: 语义搜索没有结果？

- 确认嵌入模型 `bge-m3` 已下载
- 查看设置中的嵌入统计，确认有已嵌入的笔记
- 新笔记需要等待后台嵌入处理完成

### Q: 自定义技能执行报错？

- 检查 YAML 格式是否正确
- 确认 `{content}` 占位符存在于 `user_prompt` 中
- 查看控制台日志获取详细错误信息

### Q: 嵌入管线卡住不动？

- 尝试「暂停」后重新启动
- 执行「重试失败」处理积压任务
- 如仍无效，尝试「重建全部」
