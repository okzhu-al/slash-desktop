## 全局协作定义

1. **完全直连机制**：Team 中的任何人都可以进入 Team 空间查看有权限的文件。
2. **防盗版拷贝**：Team 中的任何人查看文件时都不可以选中内容进行拷贝，在团队空间中拦截 `Cmd/Ctrl+C/A` 、右键菜单“拷贝”以及文本选中拖拽操作。
3. **隐形同步墙**：Team 中只有被授予权限的人才能在本地创建/修改文件。所有操作在文件系统的变动均会即时 Push 到团队空间。如果是通过 OS 文件管理器手工触发变动，遵循“谁 Push 上来的谁就是 `Owner` 或 `Editor`” 的认领原则。
4. **直接覆盖协作**：废弃了老旧的红绿 Diff PR 合并机制。协作者（具备写权限的成员）可直接在他人创立的文件上进行协同编辑，保存即全员同步覆盖，依靠原子时间戳（Logical Clock）与快照树（Snapshots）保证协同版本完整性。

## 🗑️ 核心删除与回收站规则 (New!)

团队协作中的数据删除极度敏感，系统通过“软删除兜底”与“极其严苛的删除特权”双轨制来治理生命周期：

1. **笔记单文件删除**：只有该笔记的 **Editor (原创建者本人)** 或全局 Admin 可以发起删除。普通协作者即便参与了成百上千次的共同编辑，也**绝对无权**软删他人创建的笔记。
2. **目录整体删除**：只有该目录的 **Owner (所有者本人)** 或全局 Admin 可以将其删除。
3. **目录删除的“紧箍咒”例外**：如果目录内**包含其他用户（Editor 为他人）创建的笔记**，或**嵌套了其他用户（Owner 为他人）的子目录**，为了保护他人数字资产，Owner 也会被**禁止删除**整个父级目录。必须先由他人移出资产后方可整端销毁。
4. **全部走废纸篓 (软删除兜底)**：平台内的所有删除指令均为“软删除 (Soft Delete)”。被斩首的目录/文件不会立刻飞灰湮灭，而是统一落入该团队的 **团队回收站 (Team Trash)**，随后由该目录的 Owner 进行“恢复 (Restore)”或“彻底物理销毁 (Hard Delete)”。
5. **根节点管网**：1 级根目录 (如 `01_PROJECTS` 等大区) 的生杀大权、及其产生的回收站条目，仅由 **Admin** 统一在 0 级维度进行管理、恢复与彻底删除。

## 角色定义

在 Slash for Team 中，系统架构分为两大原生基础角色 (`Admin` / `Observer`) 以及三大通过内容挂载的业务角色 (`Owner` / `Team-members` / `Editor`)。

### 1. Admin (最高统帅)
- **定义**：团队管理员，创建团队空间的起源。
- **核心权限**：
  1. 拥有**无限生命生杀大权**：可以任意移动、甚至直接绕过防线删除任意团队下的任何人创建的目录和笔记。
  2. 统管“1级根目录”以及其名下的团队总回收站的数据清算。
  3. 添加/开除团队成员，被添加人员默认入队身份落入 `Observer`。
  4. 可以提升任意人角色为 `Admin`。
  5. 可以为任意 Team 目录强制指派 `Team-members` 和 `Onwer`，甚至决定能否让 `Observer` 查看。

### 2. Observer (纯净看客)
- **定义**：除了 Admin，所有通过链接/邀请初进团队的人默认落在此角色。
- **核心权限**：
  1. 可以在 Team 空间云端查看被开放权限的团队文件。
  2. **无残留纯流式阅览**：本地不产生任何物理穿透文件，不进入同步轮询（Sync），只支持远程深度搜索与流式安全阅读。
  3. 在某笔记中被 `@提及` 后，会自动破格晋升为该目录的 `Team-members`。

### 3. Owner (封疆大吏)
- **定义**：管理维护自己主动 `promote to team` 到团队、或者是被 Admin 钦点接管的专属 Team 子目录。
- **核心权限**：
  1. 可以在自己管辖的目录帝国中随意招募或流放 `Team-members` 和 `Observers`。
  2. **向子赋权继承法则**：父级 `Owner` 天然继承其统辖界限内所有深层子目录的父权干预能力。
  3. **回收站裁决官**：拥有对自己目录下所产生一切废弃数据（Trash Records）的生杀予夺大权（打回人世间 / 彻底扬骨灰）。
  4. **权限边界**：出了自己管辖的目录，去往同僚的地盘，依然退化为 `Observer`。

### 4. Team-member (前沿工兵)
- **定义**：被 Admin 或 Owner 拉入战壕的协同主力。
- **核心权限**：
  1. 获取该目录下的物理文件同步权（Sync），文件将全量拉取到本机进行高优渲染。
  2. 自由协作：可在拿到权限的已有目录内**直接协同修改任何人创建的笔记**。
  3. 拓荒者转换：以 Team-member 身份如果在目录底部新建了一个全新的子目录，他将自动加冕成为那个新生子目录的 `Owner`。
  4. **删除雷区**：**永远无权删除任何非自己亲手建立的笔记或目录**。

### 5. Editor (造物神)
- **定义**：任何一篇笔记**从虚无中点击“新建”诞生的那个人**，终身绑定 `editor_id`。
- **核心权限**：
  1. 笔记本体最高管辖权：哪怕这篇文章随后加进来 100 个人进行成千上万次协同汇编，但生母永远是生母。只有 Editor 本人（及 Admin）能一键发起该独立笔记的`删除/下架`指令。
  2. （旧版审核他人权责已下乡至快照体系处理完毕）。
